What are we even looking for? The Optimization Problem

Our first order of business is actually going to be to translate this problem about data points on a plane into a network (points with connections). However, even a second of thought might give rise to the concern, “wait aren’t we losing information like that, now that we’ve just threw away the actual distance numbers for just yes’s and no’s?”.

And you’d be right, but we’re actually losing the “right kind” of information. We don’t actually want distances this would force our clustering algorithm to find a “center” and make clusters by only making circular cuts around the data points. However, network way might be more suited if you want to keep points within a cluster if they have lots of really close friends, rather than trying to find some center of a circle, which doesn’t even make sense given the shapes of our moons.

But either way, we want to encode the information into a matrix because that’s how we do fun linear algebra stuff. In fact, for networks, we can make a similarity matrices - a 2d arrays of only 0’s or 1’s, with AijA_{ij}Aij​ representing whether between nodes in the network iii and jjj are connected: a 1 for “connected”, and 0 for “not connected”.

This obviously isn’t the same as the 200-datapoint example we’re running through in the programming, but I think it’d help to reference this “by-hand” example so that we can get a better understanding of the overall setting. Consider this network of 6 points.

Then the similarity matrix would look like this:

1

And this lets us easily see that for example nodes 1 and 2 are connected, but 1 and 6 are not. (In fact, this should always be a symmetric matrix. Why?)

Neat, but that example skipped over something: how do we decide if nodes iii and jjj are connected in the first place? We have to set a thresholding value ϵ\epsilonϵ such that if points are connected if and only if they are closer than ϵ\epsilonϵ.

So the programming task seems clear: we have to somehow get all the pairwise distances, we have to get the distance from point 1 to 2, see if that’s larger or smaller than ϵ\epsilonϵ, then that decides our A12A_{12}A12​ entry. Then we compare points 1 and 3 to get our A1,3A_{1,3}A1,3​ entry, so on and so forth…

This sounds like for loop-hell, but luckily sklearn has a function that just gets all the distances for us in the correct matrix format already, and then just “threshold” by comparing each entry to ϵ\epsilonϵ (go numpy broadcasting!).

from sklearn.metrics import pairwise_distances

Measuring how we’re doing: creating the math problem

Now that we have the similarity matrix, we can start doing math on it. In particular, we want to formulate our clustering algorithm into an the language of an optimization question, because that’s all the computer’s tasks is really in machine learning.

The questions we want to answer is:

For a given network (represented by our similarity matrix where can we cut into two roughly equally-sized groupings such that we cut as few connections as possible?

And this makes sense, of course we could make the cut anywhere really, but if we have to cut a ton of connections, we’re destroying some of the clustering because those nodes we cut through must have all been really close. This minimization of cut connections is a sort of representation of cutting between two groupings that don’t have a lot of ties between each other.

Recall back to our small network example, let’s try to think about how we’d cut it.

Now if we were to answer that overarching optimization question, we could eyeball it pretty easily: just cut between nodes 2 and 4 to get two exactly equal good looking clusters.

In fact, we can see this will match our true answer, where nodes 1,2,3 belong to Cluster 0, and nodes 4,5,6 belong to Cluster 1. In fact, we have a really easy way of representing that in a vector. We can say that our “answer key”, the true clustering we’ll be aiming for is

y=[000111]y = \begin{bmatrix} 0 \\ 0 \\ 0 \\ 1 \\ 1 \\ 1 \end{bmatrix}y=⎣⎢⎢⎢⎢⎢⎢⎢⎡​000111​⎦⎥⎥⎥⎥⎥⎥⎥⎤​

So for example, y2=the cluster node 2 belongs to=0y_2 = \text{the cluster node 2 belongs to} = 0y2​=the cluster node 2 belongs to=0.

(Note that this example makes it easy to see why we need that “roughly equally sized” clause in there; otherwise we could just make a “tiny” cut to get a single node, say sever , but that wouldn’t exactly be productive.)

The Cut Term

We can define a “cut term”, which just answers part of the main question: given a guess at a clustering like above, how many “cuts” do we have to make in order to separate out the red Cluster 1 nodes from the blue Cluster 0 nodes? Like we mentioned earlier, this gives us a good idea of how entangled our clusters are, or in other words, how “bad” they are at well, clustering.

In our small example, we can come up with an idea of how to count this: you can cut every pair of mismatched nodes. So that would involve cutting between nodes 1 to 3, 2 to 3, and 2 to 4.

For our main example though, we need a programmatic way to find all the mismatched nodes. To do this, we need to find which nodes are in which cluster in the first place. If you remember back to how our labels were encoded into that y array, you can just think of this as a simple comparison to see which entries are 0 or 1.

(y==0)[:10]

array([False, False,  True,  True,  True,  True,  True,  True, False,
       False])

Of course, we want to select which indices where y[i]==0 was True, so from above we get that C0C_0C0​ has the points 6,7,9,…6,7,9,\dots6,7,9,…. To do this, we can make use of a fun little method that involes np.nonzero(), which tells us exactly what we want: the indices of array elements that are nonzero (or not false in the case of boolean arrays). For example, let’s say we want to see which points are in C0C_0C0​, or have a 0 label in y.

(nonzero returns actually a tuple, one element for each dimension of the array. y only has one dimension though, so we can just get the [0]th element of the returned tuple.)

(y==0)[:10].nonzero()[0]

array([2, 3, 4, 5, 6, 7], dtype=int64)

Once we have the points in each cluster, we have to count the number of connections between points in C0C_0C0​ and C1C_1C1​, which involves references each point in the AAA matrix. We can just add the ijijijth entry of AAA for this because, in a way, AAA is just measuring the number of connections; it’s just always 0 connections (not connected) or 1 connection (yes connected).

def cut(A,y):
    # get points in each cluster
    c1 = np.asarray(y==1).nonzero()[0]
    c0 = np.asarray(y==0).nonzero()[0]
    total = 0
    
    # count number of connections between each point
    # in c0 to points in c1
    for i in c0:
        for j in c1:
            total += A[i,j]
    return total

Now that we have our cut function, we can test it out on our optimal answer - the real labels for each moon cluster. If you remember back to our call of make_moons, we also got a numpy array called y that we can think of as our “answer key” similarly to how we had the “true labels” vector $y$ for our small example. We can see that if we cut between the two real clusters make_moons created, we don’t have to sever too many connections, as would be desired of a good clustering.

The Volume Term

With the cut term out of the way, it’s time to create the volume term. If we recall our main objective function again,

NA(C0,C1)≡cut(C0,C1)(1vol(C0)+1vol(C1))  N_{\mathbf{A}}(C_0, C_1)\equiv \mathbf{cut}(C_0, C_1)\left(\frac{1}{\mathbf{vol}(C_0)} + \frac{1}{\mathbf{vol}(C_1)}\right)\;NA​(C0​,C1​)≡cut(C0​,C1​)(vol(C0​)1​+vol(C1​)1​)

we really are penalizing small “lazy” clusters because that will make 1/vol1/\mathbf{vol}1/vol big.

Either way, to get a good Python way to calculate this using the information at hand (the similarity matrix AAA and a set of labels y that encode our C0,C1C_0, C_1C0​,C1​), we can just recall the more mathematical definition stated above:

  • vol(C0)≡∑i∈C0di\mathbf{vol}(C_0) \equiv \sum_{i \in C_0}d_ivol(C0​)≡∑i∈C0​​di​, where di=∑j=1naijd_i = \sum_{j = 1}^n a_{ij}di​=∑j=1n​aij​ is the degree of row iii (the total number of all other rows related to row iii through AAA). The volume of cluster C0C_0C0​ is a measure of the size of the cluster.

And realize that this is pretty easy to calculate using numpy.sum().

def vols(A,y):
    ds = np.sum(A, axis=1) # array of the row sums/degrees
    return (ds[y == 0].sum(), # sum of degrees in c0
            ds[y == 1].sum()) # sum of degrees in c0

With both cut and vols in hand, we can now easily recreate that normcut formula.

def normcut(A,y):
    v0, v1 = vols(A,y)
    return cut(A,y) * (1.0/v0 + 1.0/v1)

A better way to try different clusterings

Great, now that we have an objective function, we know what we need to do: the input values are the y vectors, or our guess at the clustering, and there are only finitely many choices, so why not just loop through every one to find the optimal clustering?

Now of course we could just tell a computer to have at our new function, we’re programmers/mathematicians - we can and should be a bit smarter than that, especially since the function as it stands is actually really hard to optimize. Our next move will be to define a new mathematical quantity: let z∈Rn\mathbf{z} \in \mathbb{R}^nz∈Rn such that:

zi={1vol(C0)if yi=0−1vol(C1)if yi=1z_i = \begin{cases} \frac{1}{\mathbf{vol}(C_0)} &\quad \text{if } y_i = 0 \\ -\frac{1}{\mathbf{vol}(C_1)} &\quad \text{if } y_i = 1 \\ \end{cases}zi​={vol(C0​)1​−vol(C1​)1​​if yi​=0if yi​=1​

This is just like our y label vector (zi<0  ⟺  yi=0  ⟺  i∈C0z_i < 0 \iff y_i = 0 \iff i \in C_0zi​<0⟺yi​=0⟺i∈C0​), but even better: through some fancy math, you can show that

NA(C0,C1)=2zT(D−A)zzTDz  ,\mathbf{N}_{\mathbf{A}}(C_0, C_1) = 2\frac{\mathbf{z}^T (\mathbf{D} - \mathbf{A})\mathbf{z}}{\mathbf{z}^T\mathbf{D}\mathbf{z}}\;,NA​(C0​,C1​)=2zTDzzT(D−A)z​,

where D\mathbf{D}D is the diagonal matrix with nonzero entries dii=did_{ii} = d_idii​=di​, and where di=∑j=1naid_i = \sum_{j = 1}^n a_idi​=∑j=1n​ai​ is the degree (row-sum) from before.

We can use np.where to implement the exact case behavior for this zzz.

def transform(A,y):
    v0, v1 = vols(A,y)
    
    # z_i now is the desired vector: 1/v0 if in C_0
    # and 1/v1 otherwise
    z_i = np.where(y == 0, 1/v0, -1/v1)
    return z_i

D = np.diag(np.sum(A, axis=1))
z = transform(A,y)

Asking the computer to optimize our function

At this point, basically the whole optimization problem is finally set up. Our obejctive function is now

There’s a bit of math going on for how I’m implementing this optimization in Python, but we’ll soon see a better way of doing this anyway, and the math is a bit more linear algebra than I’d like to show. def orth(u, v):
    return (u @ v) / (v @ v)*v

e = np.ones(n) 

d = D @ e

def orth_obj(z):
    z_o = z - orth(z, d)
    return (z_o @ (D - A) @ z_o)/(z_o @ D @ z_o)

Improving with some Mathemagic

Now if you’ve been playing along at home, you might have noticed that running that minimize function actually took a while, like a good 5 seconds. That time only gets exponentially worse (literally I think) as you increase the amount of data points you have. And nowadays, 200 points is a tiny tiny number, so we’re probably going to want something even faster.

Turns out, there’s even more math magic we can do to get a way faster calculation! Here’s the crux of it:

We can approximatie z_min with the eigenvector corresponding to the second smallest eigenvalue of the matrix given by L=D−1(D−A)\mathbf{L} = \mathbf{D}^{-1}(\mathbf{D} - \mathbf{A})L=D−1(D−A)

Now your first reaction might be “Woah woah, that sure came out of left field!”, and you’d be right - the amount of math needed to show this is a bit beyond the scope of this blog post. However, if you’re interested, this paper has been used a lot as a really nice and comprehensive tutorial that isn’t terribly complicated.

Either way though, if you trust that this will get us the right answer, we still have the job to do of computing this using Python. First up is constructing that matrix

L=D−1(D−A)\mathbf{L} = \mathbf{D}^{-1}(\mathbf{D} - \mathbf{A})L=D−1(D−A)

This is easy with standard numpy tools.

L = np.linalg.inv(D) @ (D - A)

And then just find the eigenvector second smallest eigenvalue of it, that’s the z\mathbf{z}z we’ve been seeking. That’s it! And we know a lot about numerical linear algebra so that finding eigenpairs is really easy nowadays.

Now the tricker part just comes from extracting the exact information we want from the format numpy gives us the eigen-stuff. The eigenvectors and eigenvalues are returned in two separate arrays, so we need an index from one to access the right vector in the other.

Wrapping up

Finally, it’s time to wrap this all in a function so that we can play around with our algorithm more easily. The inputs we had at the very beginning were of course the original data X, and the thresholding parameter epsilon. And at the end, we’ll want the “learned” labels of two clusters C0C_0C0​ and C1C_1C1​, formatted in the standard way of course.

Then, in summary, we need to

  1. Construct the similarity matrix.
  2. Construct the Laplacian matrix.
  3. Compute the eigenvector with second-smallest eigenvalue of the Laplacian matrix.
  4. Return labels based on this eigenvector.
def spectral_clustering(X, epsilon):
    """
    Given datapoint X, and a tolerance value epsilon, the maximum distance points can be apart to be considered
    "connected", returns a label array y whose elements are 0's and 1's such that the y[i] is the cluster to which
    the spectral clustering algorithm decided the X[i, :] belongs
    """
    # Part 1: Similarity matrix
    A = 1*(pairwise_distances(X) < epsilon)
    np.fill_diagonal(A, 0)
    
    # Part 2: Laplacian matrix
    D = np.diag(np.sum(A, axis=1))
    L = np.linalg.inv(D) @ (D - A)
    
    # Part 3: second-smallest eigenvalue
    vals, vecs = np.linalg.eig(L)
    idx = vals.argsort()[1]
    z_eig = vecs[:,idx]
    
    # Part 4: Turn z_eig into standard label format
    return 1*(z_eig >= 0)
